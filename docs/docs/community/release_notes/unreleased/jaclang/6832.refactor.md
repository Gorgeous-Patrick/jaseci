- **Refactor: Edge-traversal resolution now folds hop-by-hop**: `refs()` walks `ObjectSpatialPath` chains through a new `jaclang.runtimelib.resolver` whose `resolve_path_anchors` calls a uniform per-hop `resolve_hop` at every position instead of the all-or-nothing strategy decision in `planner.execute_path`. Behavior is strict parity with the previous planner: every existing bail still returns `None` and falls through to the slow edge-walk. This is the structural step toward removing that fallback (issue #6778); follow-up PRs convert each per-hop bail (predicates on non-final hops, cross-root mid-chain, etc.) into a local routing branch instead of a top-level dump. `planner.execute_path` becomes a one-line wrapper around `resolve_path_anchors` to preserve the direct test imports in `test_topology_e2e.jac`.
